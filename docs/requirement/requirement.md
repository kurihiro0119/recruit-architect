# 1. システム概要（目的・背景）

## 目的

Bizdev 採用に関する以下の情報を**スプレッドシートに依存せず一元管理**し、

- 情報の分散・属人化を防ぐ
- 状況把握・更新を容易にする
- 将来的なデータ活用（分析・可視化）を可能にする

ことを目的とする。

## 対象業務

- 採用 KPI 管理
- 採用施策管理
- 企業・競合分析
- 求人票管理
- 業務内容定義
- 他社求人リサーチ
- 組織・体制管理
- 選考プロセス管理
- 採用チャネル管理
- FAQ 管理

# 3. 機能要件（大枠）

## 3.1 共通機能

- CRUD（作成・閲覧・更新・削除）
- 履歴管理（いつ・誰が・何を変更したか）
- コメント / メモ機能
- 権限管理（閲覧のみ / 編集可）

# 4. 機能要件

## 4.1 KPI 管理機能（KPI について）

### 主なデータ

- 期間（年月 / フェーズ）
- KPI 種別
  - 人数・歩留まりKPI
  - タイムラインKPI
- 目標値
- 実績値
- 差分（自動計算）

### 機能

- KPI の期間別管理
- 自動集計・差分表示
- グラフ表示（将来拡張）

## 4.2 採用施策管理（全体施策）

### 主なデータ

- 時期
- 日程
- マイルストーン
- 主担当
- アウトプットとゴール

### 機能

- 時系列（タイムライン）
- 担当者アサイン

## 4.4 求人票管理（求人修正案 + 求人マスタ）

### 主なデータ

- 求人 ID
- ポジション名
- 募集背景
- 業務内容
- 必須 / 歓迎条件
- 修正案（複数案）

### 機能

- 求人単位での情報集約
- 修正履歴管理
- ステータス管理

## 4.5 業務・役割定義管理（業務詳細系）

### 主なデータ

### 機能

- 表形式表示
- 求人票との紐付け（将来）

# 5. 画面要件（最低限）

- **ダッシュボード**
  - KPI サマリ
  - 進行中施策
- **一覧画面（表）**
- **詳細 / 編集画面**
- タブ or サイドナビでシート相当を切替

# 6. データ要件（設計方針）

- 「スプレッドシートの列 = DB カラム」にしない
- 概念単位で正規化
  - 求人
  - KPI
  - 施策
  - マスタ（職種・チャネルなど）

# 7. 非機能要件

- 権限管理（閲覧/編集）
- 変更履歴の保持
- 将来の機能追加を前提とした拡張性
- スプレッドシートからの移行を考慮
